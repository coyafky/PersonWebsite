# 2026-W14

第 14 周周志

## 🎯 本周待办（MIT）

- 明确深土生图/车膜预览项目的技术路线：比较高清拍摄、商家电子图、色差仪 L/A/B 数值三种方案。
  
- 推进 Hermes Agent 云端化方案，用腾讯云服务器替代 OpenClaw 部署思路，实现 24 小时不断档运行。
  
- 探索“飞书妙记录音 -> Lark CLI 转文字 -> Hermes Skill 生成博客 -> GitHub/Vercel 自动推送”的表达能力增强工作流。
  

### 已完成

- 针对生图项目形成三种方案判断：高清拍摄后渲染、基于商家电子版图片渲染、使用色差仪获取精确颜色数值。
  
- 在生图项目中初步确定优先采用色差仪获取 L/A/B 数值，再做线上预览的技术路线。
  
- 判断高清拍摄或原始图车辆渲染即使使用最新 ChatGPT 模型仍存在误差，不适合作为当前主方案。
  
- 明确商家缺少可直接用于渲染的电子版图片，因此第二种方案暂时不可行。
  
- 梳理 Hermes Agent 替代 OpenClaw，并部署到腾讯云服务器以实现 24 小时运行的方向。
  
- 参考并收藏 LucasBlog 仓库，作为后续博客自动化写作和发布系统的框架参考。
  

### 进行中

- 生图色差仪方案仍处于项目立项阶段，需要继续验证设备成本、采集流程和线上预览效果。
  
- 销售 Agent 方向仍在探索中，计划基于 Book/Bookless（名称待确认）能力对销售材料做结构化处理。
  
- 飞书妙记到博客自动发布的工作流还需要优化，包括转写质量、博客格式规范、Hermes Skill 输出稳定性和自动推送链路。
  

## 💡 思考（Insights）

- 对生图/车膜预览类项目来说，“真实颜色”比“看起来很高清”更关键。拍摄设备越好只能减少图像噪声，但不能完全消除光照、材质和模型渲染带来的色差。
  
- 当高清原始图和电子版图片都无法满足稳定交付时，使用色差仪获取 L/A/B 数值虽然成本更高，但更接近可控、可验证的工程路线。
  
- Hermes Agent 的价值不只是本地辅助，而是可以通过云端部署变成 24 小时在线的工作流基础设施。
  
- 表达能力可以被系统化增强：先用飞书妙记降低记录成本，再用 Lark CLI 和 Hermes Skill 把口述内容转成结构化博客，最后接入 GitHub/Vercel 自动发布。
  
- 博客系统不只是输出文章的地方，也可以成为个人思考、项目复盘和技术资产沉淀的长期容器。
