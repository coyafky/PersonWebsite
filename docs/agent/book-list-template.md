# Book List 模板

> Hermes 在 `/book-list-from-inbox` 时使用此模板生成 `content/book-list/<date-slug>.md`。

## 模板

```markdown
---
title: ""              # 读完后给这本书起的中文短名
date: ""               # 读完日期 (YYYY-MM-DD,**必须加引号**避免被解析为 Date 对象)
summary: ""            # 1-2 句核心收获
status: draft          # ⚠️ 永远默认 draft
author: ""             # 必填,作者全名
genre: ""              # 必填,单字段类别(如"认知科学"/"工程实践"/"商业思维")
tags:                  # 2-5 个细标签
  - ""
lang: zh
englishSummary: ""     # 1-2 句英文
---

## 这本书讲了什么

## 我学到了什么

## 我会怎么用

## 引用与摘录
```

## 字段语义

### 必填

- `title` — 读完后给这本书起的中文短名(≤20 字,避免"《》"书名号)
- `date` — **读完**日期(非购买日、非开始日)
- `summary` — 1-2 句中文摘要,讲清楚**为什么值得读**
- `status` — **必须 `draft`**,由 Coya 手动改为 `published`
- `author` — 作者全名,中文写中文名,英文写英文名
- `genre` — 单字段粗分类,见下枚举
- `tags` — 细粒度标签(2-5 个,如 `["注意力", "深度工作", "Tim Ferriss"]`)

### 可选

- `lang` — 默认 `zh`,英文书写 `en`
- `englishSummary` — 1-2 句英文摘要

## `genre` 枚举(建议,可不限于此)

| 值 | 含义 |
|----|------|
| `认知科学` | 思维、学习、记忆、注意力 |
| `工程实践` | 软件工程、SRE、系统设计 |
| `商业思维` | 商业模式、产品策略、谈判 |
| `人物传记` | 创业者、科学家、艺术家 |
| `历史社会` | 历史、社会学、人类学 |
| `文学小说` | 小说、散文、诗 |
| `哲学思辨` | 哲学、伦理、存在主义 |
| `科普知识` | 数学、物理、生物、跨学科 |
| `职业发展` | 求职、领导力、沟通 |
| `个人成长` | 习惯、情绪、健康、关系 |
| `设计美学` | 设计、艺术、视觉 |
| `其他` | 兜底 |

> 不要过度细分 — `genre` 是首页筛选用的粗粒度,1 本书只对应 1 个 genre。

## `tags` 写法

- 用**概念**而非**形容词**(`注意力` 而不是 `重要的`)
- 用**通用术语**而非**个人代号**(`认知偏差` 而不是 `我最近在想的事`)
- 中英文混排时优先用英文(`"feedback loop"`, `"systems thinking"`)

## 正文 4 段写法

### 这本书讲了什么

- 2-4 段,客观描述书的核心论点
- 不要写"本书共 X 章,作者是 Y"
- 重点是**观点结构**,不是目录

### 我学到了什么

- 3-5 条 Coya 第一人称的 takeaways
- 来自书的事实直接写,Hermes 推断的标 `[待确认]`
- 每条要能独立成段,可以"我倾向于…"开头

### 我会怎么用

- 1-3 条**具体可执行**的改变
- "我下周开始…" / "下次做 X 时我会…"
- 不要写空话("我会更努力" / "我会更珍惜时间")

### 引用与摘录

- 1-5 条原文摘录,标注章节或页码
- 只摘**值得反复看**的句子
- 不摘"目录"和"开头引言"

## 与 `summary` 的区别

- `summary` = 摘要(给列表页 + SEO 用,1-2 句)
- 正文"这本书讲了什么" = 内容(给详情页用,2-4 段)
- 不要把 `summary` 复制到正文
