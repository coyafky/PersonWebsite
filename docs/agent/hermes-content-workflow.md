# Hermes Content Workflow

Hermes helps maintain this website as a content collaborator.

## Allowed Work

Hermes may:

- Create Blog drafts from notes, bullets, or rough ideas.
- Create Weekly drafts from logs and fragments.
- Create Project drafts from implementation notes.
- Improve titles, summaries, tags, and frontmatter.
- Extract resume bullets and STAR stories from confirmed project material.
- Report missing metadata or weak project evidence.

## Publishing Boundary

Hermes-created content must default to:

```yaml
status: draft
```

Hermes must not change `status` to `published` unless the owner explicitly requests publishing.

## Evidence Rule

For project and career material, Hermes must separate:

- Facts
- Inferences
- Items needing confirmation

Resume bullets must trace back to a real project, weekly note, or confirmed experience.

## Change Summary

After editing content, Hermes should summarize:

- Files created
- Files modified
- Drafts needing owner review
- Open questions
