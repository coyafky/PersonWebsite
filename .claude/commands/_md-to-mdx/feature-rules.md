# Feature Detection Rules — MD-to-MDX Skill

每篇文章被扫描后，会按下面的规则识别**时间线 / 对比 / 关键指标 / 决策流**四种特征。命中哪个特征，就从 `templates/` 拷贝对应的模板并填数据。

> 这些规则是**启发式**的——目标是减少误报，宁可漏报也不要硬转。识别结果要先给用户看，由用户确认后再实例化组件。

---

## 1. 时间线（Timeline）

**触发信号（满足任一即视为命中）：**

- 标题含 `## Time Log` / `## 时间线` / `## 日程` / `## Timeline`
- Markdown 表格 ≥3 行，且第一列是日期或星期（如 `**周一 5/18**` / `2026-05-19`）
- 一组带 `**` 加粗前缀的列表项，每个对应一个日期

**典型来源：**

```md
| 日期 | 活动 | 项目标签 |
|------|------|----------|
| **周一 5/18** | 安装 skills | `#Hermes技能` |
| **周二 5/19** | Memory 系统设计 | `#共享记忆` |
```

**推荐模板：** `templates/timeline.template.tsx`

**抽取规则：**

- `label` ← 表格第一列（日期）
- `title` ← 第二列（活动）
- `tags` ← 第三列（如有，按空格分割多个 hashtag）

---

## 2. 对比（Comparison）

**触发信号（满足任一即视为命中）：**

- 标题含 `对比` / `比较` / `vs` / `VS` / `A vs B` / `走法对比`
- 两张或更多相邻 Markdown 表格，列数相同（典型 3 列：方案 / 优点 / 缺点）
- 一个候选方案列表，每项紧跟一个子项目符号列表

**典型来源：**

```md
| 方案 | 优点 | 缺点 |
|------|------|------|
| 走法 A | 文件少 | 仍依赖 prop |
| 走法 B | 绕开 RSC 序列化 | 组件文件多 |
```

**推荐模板：** `templates/comparison.template.tsx`

**抽取规则：**

- 每个表格行 → 一个 card
- `title` ← 第一列（方案名）
- `meta` ← 从标题提取（如 `走法 A` → `走法`）
- `pros` ← 第二列（按 `,` 或 `、` 分割）
- `cons` ← 第三列（同上）

---

## 3. 关键指标（Stats）

**触发信号（满足任一即视为命中）：**

- ≥3 行内容，形如「`<大数字><单位>` + `<描述>`」
  - 例子：`180 条 skill` / `28KB+ 笔记` / `95KB+ 笔记` / `1941x`（性能对比）
- 一个 bulleted list，每个 item 以数字开头
- `Highlights` / `产出` / `成绩` 段落含 ≥3 个加粗数字

**推荐模板：** `templates/stats.template.tsx`

**抽取规则：**

- `value` ← 数字部分（保留 `+` / `x` / `KB` / `%` 等单位符号）
- `unit` ← 数字后的中文量词（可选）
- `label` ← 紧随其后的描述短语

---

## 4. 决策流（Flow）

**触发信号（满足任一即视为命中）：**

- ```` ```mermaid ```` 围栏（已存在的流程图）
- 标题 `## 流程` / `## 步骤` / `## 决策树` / `## 走法` + 后续有序列表含分支语言
- 文字步骤里出现 `如果 X → A；否则 → B` / `第一步 / 第二步 / 第三步` / `分支 A / 分支 B`

**推荐模板：** `templates/flow.template.tsx`

**抽取规则：**

- 如果已有 mermaid 围栏：直接转写为 mermaid 字符串（去掉围栏），灌入模板
- 如果是文字流程：让 skill 帮你生成 mermaid 语法（`graph LR` / `graph TD`），用户确认后再灌入
- ⚠️ **必须**用普通双引号字符串 `chart="..."`，**禁止**模板字面量 `chart={\`...\`}`（RSC 序列化会丢弃）

---

## 不命中的情况

如果一篇文章四种特征都未命中（普通叙述型博客、纯观点类文章），**不强转**。skill 应在候选清单中标 `no-feature-matched`，询问用户是否要强制升级为 MDX（仅做格式迁移、不生成组件）。
